# Project Field Hub Pro Walkthrough Video

Target length: 3 to 4 minutes

Audience: clients, contractors, subcontractors, consultants, and admins who need to understand the basic project workflow.

Recording format: 16:9 desktop screen recording with voiceover. Use a clean demo project and keep the browser zoom at 100%.

## Video Structure

### 1. Opening

Duration: 15 seconds

On screen:
- Open the dashboard.
- Show the top navigation, active project card, dark mode toggle, and recent notifications.

Voiceover:

Welcome to Project Field Hub Pro. This dashboard keeps project records, submissions, reports, financial items, defects, and close-out tracking in one shared workspace. Everyone assigned to the same project can see the updates relevant to their access.

### 2. Switch Theme

Duration: 15 seconds

On screen:
- Click Dark mode in the top navigation.
- Click Light mode to switch back.

Voiceover:

You can switch between light mode and dark mode from the top navigation. The app remembers your choice, so the next time you open the dashboard it loads in your preferred mode.

### 3. Create Or Select A Project

Duration: 25 seconds

On screen:
- Show the Projects section.
- Point out project chips.
- Briefly show the project creation form.

Voiceover:

At the top of the dashboard, you can create a new project or switch between projects you have access to. Each project has its own timeline, team setup, submissions, reports, financial records, and notification history.

### 4. Review Recent Notifications

Duration: 25 seconds

On screen:
- Show the Recent project changes panel.
- Highlight actor, section, title, details, and timestamp.

Voiceover:

The notification panel shows recent changes made inside the active project. When a team member creates, updates, deletes, approves, rejects, or imports records, the update is logged here so the rest of the project team knows what changed.

### 5. Navigate Modules

Duration: 25 seconds

On screen:
- Use the left project menu.
- Click Overview, Documents Submission, Daily Reports, Financials, and Defects.

Voiceover:

The project menu lets you move between modules. Access is role-based, so each user only sees the modules they are allowed to use.

### 6. Update Project Overview

Duration: 25 seconds

On screen:
- Open Overview.
- Expand Project details.
- Show fields for project name, location, client, contractor, handover date, completion date, and details.

Voiceover:

The Overview module stores the core project information. Updating these details refreshes the dashboard summary and creates a notification for users on the same project.

### 7. Manage Team Information

Duration: 25 seconds

On screen:
- Open Contractor information.
- Show company name, type, and responsible trades.
- Open Consultant details.

Voiceover:

The team setup areas keep contractor and consultant information organized. Main contractors, subcontractors, architects, and MEP consultants can be listed with their responsible trades.

### 8. Submit And Review Documents

Duration: 35 seconds

On screen:
- Open Documents Submission.
- Show contractor submission form.
- Show consultant document form.
- Show approval or return controls.

Voiceover:

The submission module handles contractor submissions, RFIs, method statements, project programmes, and consultant documents. Depending on the user role, submissions can be created, reviewed, accepted, returned, or reset to pending. Review decisions are also logged in notifications.

### 9. Add Reports

Duration: 30 seconds

On screen:
- Open Daily Reports.
- Show date, location, work done, manpower, and attachment fields.
- Open Weekly Reports.

Voiceover:

Daily and weekly reports capture site progress. You can add descriptions, manpower notes, progress summaries, and supporting attachments so the project history stays complete.

### 10. Track Financials

Duration: 30 seconds

On screen:
- Open Financials.
- Show quotation, invoice, or variation order entry.
- Show status cards and client review actions.

Voiceover:

The Financials module tracks quotations, invoices, and variation orders. Submitters can prepare and submit records for review, while authorized client-side users can approve, reject, or mark items as paid.

### 11. Record Completion And Defects

Duration: 35 seconds

On screen:
- Open Completion.
- Show checklist entry.
- Open Defects.
- Show defect zone, title, status, details, photo upload, and Excel import.

Voiceover:

Completion and Defects help manage close-out. You can create checklist items, add defect records with photos, organize defects by zone, and import defect registers from Excel.

### 12. Manage Access

Duration: 25 seconds

On screen:
- Open Access Control.
- Show email, role, module permissions, and save button.

Voiceover:

Admins can assign users to a project and control which modules they can access. This keeps each party focused on the records they are responsible for.

### 13. Closing

Duration: 15 seconds

On screen:
- Return to dashboard overview and notifications.

Voiceover:

That is the core workflow: create a project, invite the right users, update records through each module, and use notifications to keep the team aligned as the project changes.

## Recording Checklist

- Use a demo project with sample data.
- Hide unrelated browser tabs.
- Keep browser zoom at 100%.
- Record in light mode first, then show the dark mode toggle early.
- Keep mouse movement slow and intentional.
- Pause briefly after every click so viewers can understand the screen.
- Avoid entering real client, project, email, or financial data.

## Suggested Chapters

1. Dashboard and Theme
2. Projects and Notifications
3. Modules
4. Submissions and Reviews
5. Reports and Financials
6. Completion, Defects, and Access
